OMNA Presence and service-description Registry

OMNA maintains this registry to record values used by known services when invoking the OMA-defined Presence enabler.

The <service-description> element is defined in the OMA-TS-Presence_SIMPLE-V1_1 specification. This element is composed of three subordinate element: <service-id>, <version> and <description>. Each unique instance of the <service-id> and <version> is expected to be recorded in this table. The <description> value is optional and may be set by the application without regard for the values shown inthe examples below.

New entries to be added to this table must be reviewed with the OMA COM WG. They will assure that the required data is correctly structured and does not conflict with existing service descriptions.

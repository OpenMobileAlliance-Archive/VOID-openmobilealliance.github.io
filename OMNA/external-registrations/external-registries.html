OMNA Assignments from External Registries
This information describes values and assignments that have been recorded by external registries in support of OMA activities. The registries include:
  • Internet Assigned Numbers Authority (IANA)
  • Joint ISO/ITU-T

This is merely intended to provide informative tracking for such assignments and should not be viewed as an exclusive or complete list of value assignments received.

References to the associated codes or tables are provided as applicable.

OMA Object Identifier (OID)

OMNA is continuing to utilize the assignment by the Joint ISO/ITU-T Object Identifier made to the WAP Forum. This assignment permits OMA to manage subordinate assignments.

The OMA root Object Identifier is

{joint-iso-itu-t(2) international-organizations(23) wap(43)}

Assignments associated to this object identifier can be located on the OMNA ISO Object Identifier page.

OMA Uniform Resource Name (URN)

OMNA has registered a formal Uniform Resource Name (URN) with IANA. This was done using the proscribed procedures resulting in the formal registration in RFC 4358 and recorded on the IANA Formal URN Namespace Registry.

The OMA URN Root is

urn:oma:

Assignments utilizing the OMA URN are recorded on the OMNA URN Namespace Registry.

OMA Private Enterprise Number

OMNA has reserved a Private Enterprise Number with IANA to permit it to define private Attribute-Value Pair (AVP) codes for use in Diameter applications.

The OMA Private Enterprise number is

30079

This value has been recorded on the IANA Private Enterprise registration page.

The value is the basis for OMNA Registry for OMA Private AVP Codes.

Application IDs

IANA records Application Ids for use in Diameter applications. OMA applications can reserve specific values.

Application Id reserved for OMA defined use is

Policy Processing 16777243 (0x100001B)

The registration can be located on the IANA AAA Parameters Page.

Application Provider Identifier

OMA has registered an Application Provider Identifier per the ISO/IEC 7816-5 Registration Authority. This will permit OMNA to provide Application Identifier (AID) codes for card applications by assigning Proprietary Application Identifier Extension (PIX) codes to differentiate different international IC card applications associated with OMA defined services.

The OMA Registered Application Provider Identifier (RID) is

A000000412

DVB Service Information - CA System ID

The Digital Video Broadcasting (DVB) Project provides a registry for values that are communicated over DVB systems. OMA has registered and received an assignment for a CA System ID value. This value is used for the CA_system_ID field of CA Descriptor of the DRM Content Format for MPEG-2 Profile OMA-TS-SCE_MDCF-V1_0.

The CA System ID registered for OMA is

0x0004

The DVB registrations can be reviewed at DVB Identifiers Registry for CA System ID.

OMA PUSH Application ID
OMNA assigns numbers for registered PUSH Application IDs. The number space is divided into three segments:
Number Space Segments
Range Description
0x0000 to 0x7FFF Well-Known values are set as specified by the OMA working group processes. The application as represented by the URI must be defined in an associated OMA specification. URNs are required for URI and NID in URN [ rfc2141 ] MUST be 'x-oma-application'.
0x8000 to 0xFF000000 Reserved values, registered and managed by OMNA. URN with NID 'x-wap-application' cannot be used.
0x4000 to 0x1FFFFF Experimental values, registered by OMNA only for advisory purposes. URIs in the Well-Known and Reserved categories SHOULD NOT be used.
In order to register a PUSH Application identifier, a short description of its use must be provided. The information to be included in a request may be found here. See WAP-189 PUSH OTA Specification for more information on PUSH Application identifiers.

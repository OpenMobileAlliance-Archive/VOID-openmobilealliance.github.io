OMNA Service and Policy exception codes registry for OMA RESTful Network APIs
This registry stores the exception codes for Service Exceptions and Policy Exceptions of the OMA RESTful Network APIs.

The exception codes from 0001 to 0999 have been inherited from ParlayX [PSA1.0] and ParlayREST[ParlayREST2.0]. New exception codes defined by the individual OMA RESTful Network APIs occupy the range from 1000 to 1999.

New common exception codes are defined in the range from 2000 to 2099. The range from 2100 to 2999 is reserved for future use. The range from 3000 to 3499 can be used for experimental or private purposes; these values will not be assigned in a specification. The following assignment policy applies to this registry:
  • New assignments to be requested by eMail to OMNA
  • The content for 'Meaning' and the link to the affected specification needs to be provided.
  • The desired code range (1000+ or 2000+) needs to be indicated.
  • This registry is restricted to OMA specifications only
  • The request will be forwarded by OMNA to the Architecture WG for review
  • New assignments will receive an unassigned number (usually the lowest) in the appropriate range. Only numbers from ranges open for assignment will be assigned.
